# Security Design

## Overview

This project implements a **defense-in-depth** security model with multiple layers of protection. No single control failure compromises the entire system.

## Network Security

### Security Group Rules Matrix

#### Bastion Security Group (`bastion-sg`)

| Direction | Port | Protocol | Source/Destination | Purpose |
|-----------|------|----------|-------------------|---------|
| Ingress | 22 | TCP | Admin CIDR (configurable) | SSH access |
| Egress | All | All | 0.0.0.0/0 | Outbound traffic |

> ⚠️ **Production Requirement**: `admin_cidr_blocks` must be set to your office/VPN IP. Never use `0.0.0.0/0` in production.

#### Application Security Group (`app-sg`)

| Direction | Port | Protocol | Source/Destination | Purpose |
|-----------|------|----------|-------------------|---------|
| Ingress | 80 | TCP | 0.0.0.0/0 | HTTP traffic |
| Ingress | 443 | TCP | 0.0.0.0/0 | HTTPS traffic |
| Ingress | 22 | TCP | Bastion SG | SSH from bastion only |
| Egress | All | All | 0.0.0.0/0 | Outbound traffic |

#### Database Security Group (`db-sg`)

| Direction | Port | Protocol | Source/Destination | Purpose |
|-----------|------|----------|-------------------|---------|
| Ingress | 3306 | TCP | Application SG | MySQL from app tier |
| Ingress | 3306 | TCP | Bastion SG | MySQL admin from bastion |
| Egress | All | All | 0.0.0.0/0 | Outbound traffic |

### Network Isolation

- **Public subnets**: Only bastion host and NAT gateway
- **Private subnets**: Application servers and RDS — no public IP, no direct internet access
- **RDS**: `publicly_accessible = false` — only reachable within VPC

## Secrets Management

### What's Protected

| Secret | Protection Method |
|--------|-------------------|
| DB password | Terraform variable marked `sensitive`, `.tfvars` in `.gitignore` |
| AWS credentials | AWS CLI profile or OIDC (CI/CD) — never in code |
| IAM user passwords | Generated by Terraform, output marked `sensitive` |
| SSH keys | `.pem` and `.key` extensions in `.gitignore` |
| State file | Encrypted S3 bucket, not in version control |

### What's NOT in Version Control

The `.gitignore` excludes:
- `*.tfvars` — Contains environment-specific secrets
- `*.tfstate` / `*.tfstate.*` — Contains all resource attributes including secrets
- `*.pem` / `*.key` — SSH private keys
- `.terraform/` — Provider binaries and cached modules

## IAM Security

### Policy Design

| Entity | Policy | Scope | Rationale |
|--------|--------|-------|-----------|
| Developer group | S3 Read | `arn:aws:s3:::aws-infra-platform-*` | Scoped to project buckets, not `*` |
| Independent user | EC2 Read-Only | `ec2:Describe*`, `ec2:Get*`, `ec2:List*` | Read-only, not `ec2:*` |

The original lab code used `"Resource": "*"` with `ec2:*` — this violates least privilege. The production version scopes S3 access to project-prefixed buckets and limits EC2 to read-only actions.

### User Lifecycle

- Login profiles have `password_reset_required = true`
- In dev/staging, `force_destroy = true` allows clean teardown
- In prod, `force_destroy = false` prevents accidental user deletion

## EC2 Hardening

| Control | Setting | Purpose |
|---------|---------|---------|
| IMDSv2 | `http_tokens = "required"` | Prevents SSRF attacks from stealing instance metadata |
| Encrypted EBS | `encrypted = true` | Data at rest encryption |
| No public IP (app) | Private subnets | No direct internet exposure |

## RDS Security

| Control | Dev | Staging | Prod |
|---------|-----|---------|------|
| Encrypted storage | ✅ | ✅ | ✅ |
| Multi-AZ | ❌ | ❌ | ✅ |
| Deletion protection | ❌ | ❌ | ✅ |
| Final snapshot | Skip | Skip | Required |
| Backup retention | 1 day | 3 days | 7 days |
| Publicly accessible | ❌ | ❌ | ❌ |

## State File Security

| Control | Implementation |
|---------|---------------|
| Storage | S3 bucket with versioning |
| Encryption | AES-256 server-side encryption |
| Access | Bucket policy restricts to specific IAM roles |
| Public access | All 4 public access blocks enabled |
| Locking | DynamoDB table prevents concurrent modifications |
| Version history | S3 versioning allows state recovery |
